# Security & Code Audit — 2026-04-01

Fifteen automated agents audited the full codebase across security, performance,
API design, data model, configuration, UX, usability, feature gaps, and PAM/SSSD
integration. Findings are summarized below by category and severity.

---

## Security

### Critical / High

- **One-tap GET approval** (`handlers_mutations.go:88`): `GET /api/onetap/{token}`
  approves a sudo challenge. Slack/Discord/iMessage link previewers auto-fetch GET
  URLs and silently approve challenges before the user sees them. Fix: GET renders
  a confirmation page; POST performs the approval.

- **Admin role never re-validated** (`session.go:260`): Role is baked into the
  session cookie and never checked against PocketID again. `verifyFormAuth` actively
  re-issues the admin cookie on every mutation, extending the window indefinitely.
  A demoted admin retains full access until they close their browser.

- **`IDENTREE_DEV_LOGIN` checkbox in admin UI** (`handlers_admin.go:497`): Any admin
  can enable this from the config page, save it to TOML, and it survives restarts —
  giving `GET /dev/login?user=alice&role=admin` to anyone who reaches the server.
  Must be removed from the config form entirely.

- **`GET /signout` is CSRF-vulnerable** (`server.go:255`): `<img src="/signout">`
  from any third-party page silently signs out the user. Must be `POST`.

- **`oneTapUsed` not persisted** (`challenge.go`): Replay protection tokens are
  in-memory only. After a restart any issued one-tap URL can be replayed.

- **`revokeTokensBefore` lost without `SessionStateFile`** (`config.go`): Revocations
  are silently forgotten on restart. No startup warning when `SessionStateFile` is
  absent.

### Medium

- **Revocation race** (`challenge.go`): `Approve()` doesn't check whether
  `revokeTokensBefore[username]` was set after the challenge was created. A challenge
  created before a revocation can be approved afterward, creating a new grace session.

- **`SharedSecret == ""`** (`handlers_api.go:31`): Opens all machine API endpoints
  (`/api/challenge`, `/api/grace-status`, etc.) unauthenticated with only a log
  warning rather than a startup failure.

- **CSRF token not bound to session instance** (`session.go:204`): `computeCSRFToken`
  is HMAC over `username:timestamp` with no session binding.

- **`verifyAPIKey` timing leak** (`handlers_api.go:74`): `subtle.ConstantTimeCompare`
  returns 0 immediately on length mismatch. Hash both sides before comparing, as
  `verifySharedSecret` already does.

- **`IDENTREE_LDAP_DEFAULT_HOME` format string injection** (`ldap.go:274`):
  `fmt.Sprintf(homePattern, username)` treats operator config as a Go format string.
  Use `strings.Replace`.

- **Unsigned flash cookie** (`session.go:153`): `config_error:` flash embeds raw
  `err.Error()` containing `:` and `,` which breaks the delimiter-split parser. URL-
  encode error strings in flash cookies.

- **`install.sh` shell injection** (`handlers_install.go:362`): `text/template` embeds
  `ServerURL` raw into a shell script. Metacharacters in the URL inject shell commands.

- **Partial PocketID group fetch silently replaces good LDAP data** (`pocketid.go:216`):
  A transient 429 on one group's detail endpoint drops that group's sudo permissions
  for all members.

- **Empty `groups` OIDC claim silently demotes admins** (`handlers_oidc.go:188`):
  No error logged; user gets `role="user"` with no indication why.

- **HKDF `nil` salt for escrow key derivation** (`escrow.go:67`): RFC 5869 recommends
  a random salt; low-entropy operator keys are weakened without one.

- **`LDAP_BIND_PW` unquoted heredoc** (`entrypoint.sh:85`): A `#` in the password
  becomes a comment; a newline injects config lines into `sssd.conf`.

### Low

- Legacy 3-part session cookie accepted indefinitely with no cutover date (`session.go:79`).
- CSRF window is ±5 min (10 min total) due to `.Abs()` — should be unidirectional (`session.go:248`).
- Manual XOR constant-time comparison instead of `subtle.ConstantTimeCompare` (`util.go:114`).
- Unauthenticated `/api/webhook/pocketid` when `WebhookSecret` unset — continuous LDAP refresh DoS (`server.go:178`).
- Any API key grants full all-users history export with no scoping (`handlers_dashboard.go:1439`).

## Performance & Concurrency

- **`reap()` holds global write lock during disk I/O** (`challenge.go:1021`): Full
  `json.Marshal` + `fsync` + rename happen under `mu.Lock()`. Extract I/O after lock.

- **`LogAction` syncs to disk on every call** (`challenge.go:619`): Write-back buffer
  with dirty timer (e.g. 2s) would eliminate most syncs.

- **`ActiveSessions(username)` full map scan** (`challenge.go:500`): String prefix
  scan over all `lastApproval` keys. Use nested map structure.

- **`KnownHosts(username)` full log scan per page load** (`challenge.go:712`): Build
  a live side-index updated in `LogActionAt`.

- **O(U) `ActiveSessions` calls in admin access page** (`handlers_dashboard.go:757`):
  Call `AllActiveSessions()` once, index locally.

- **`broadcastSSE` holds `sync.Mutex` while iterating channels** (`sse.go:38`):
  Copy channel list under lock, release, then send. Use `sync.RWMutex`.

- **`crypto/rand.Read` on every HTTP request** (`server.go:311`): Including `/healthz`
  and `/metrics`.

- **History fully allocated then sliced to `[:5]`** (`handlers_dashboard.go:256`):
  Pass a `limit` parameter to `ActionHistory`.

## API Design

- Verb-tunnelled URLs: `/api/challenges/approve`, `/api/sessions/revoke-all`,
  `/api/sudo-rules/add`, `/api/hosts/remove-host`, `/api/users/remove`, etc.
  These should use HTTP methods on resources (`DELETE /api/sessions/{user}/{host}`).

- `GET /theme?set=dark` mutates state — link previewers can silently change user themes.

- `/api/challenge` (singular) vs `/api/challenges/` (plural) — inconsistent.

- `revokeErrorPage` (HTML) called from `/api/` endpoints — API routes must return JSON errors.

- `/api/admin/users/claims-json` — routing by URL suffix instead of HTTP method.

- Webhook returns `401` for bad HMAC, should be `403`.

---

## Data Model & Storage

- No version field in JSON state file — migrations are ad-hoc with no schema detection.

- Grace key `username@hostname` breaks with `@`-containing usernames.

- `lastOIDCAuth`, `oneTapUsed`, `revokeTokensBefore`, `rotateBreakglassBefore` grow
  without bound — no reaper coverage.

- Dashboard session cookie has no server-side revocation list.

- `LDAP dir == nil` at startup served as `ResultBusy` with no retry backoff.

---

## Configuration & Deployment

- **`/healthz` is a hardcoded `"ok"` stub** — always 200 regardless of LDAP/PocketID
  state. Container orchestration never restarts a broken instance.

- **`LDAPUIDBase` documented default is 200000, actual default is 0** (UID 0 = root).

- **`IDENTREE_HISTORY_PAGE_SIZE` (UI/TOML) ≠ `IDENTREE_PAGE_SIZE` (config loader)**:
  setting applies in memory until restart then silently vanishes.

- **`IDENTREE_OIDC_INSECURE_SKIP_VERIFY` uses `== "true"` not `getBool()`** — `"1"`,
  `"yes"` silently evaluate to false.

- **Dockerfile missing `/config` directory** — default data paths fail without a volume.

- 94 `log.Printf` + 31 `slog` calls coexist — mixed log stream breaks aggregation.

- `IDENTREE_ESCROW_VAULT_MAP` JSON parse failure silently ignored.

- `ldap_tls_reqcert = never` hardcoded in `entrypoint.sh` with no override path.

---

## Dumb/Inconsistent Design Choices

- Action log action strings (`"approved"`, `"auto_approved"`, `"rotated_breakglass"`,
  `"removed_host"`) are untyped raw string literals copy-pasted across 5 files. A typo
  creates a log entry that renders incorrectly.

- `IssuerPublicURL` rewriting uses `strings.Replace` on the raw URL string — could
  match a substring in a query parameter.

- `ValidateHost` returns `true` when the registry is empty ("backward compat" landmine).

- `LDAPSudoNoAuthenticate` is an untyped string with 3 valid values instead of an enum type.

- `ExtendGraceSession` 75% guard silently no-ops with misleading "extended" log entry.

---

## Functionality Gaps

- **No all-users pending queue for admins** — `AllPendingChallenges()` exists in the
  store but is never surfaced in the UI. Critical for `AdminApprovalHosts` deployments.

- **`revealed_breakglass` sends no webhook notification** — highest-severity action
  has no external signal.

- **Config/claims/sudo-rule changes not in the action log** — silent admin privilege
  escalation leaves no audit trail.

- **LDAP user deletion not propagated to the LDAP directory** — periodic refresh
  restores deleted users from PocketID.

- No time-based access policies, no approval delegation, no session scheduling.

- `add-host` and `rotate-host-secret` are CLI-only.

## Usability & UX

- PocketID down = silent empty permissions panel — no banner, no explanation.
- LDAP sync failures invisible to operators (logged only).
- PAM client shows same "Expired." for timeout, server unreachable, and missed window.
- Rate-limit error surfaces as raw HTTP string at the terminal.
- `not_authorized_message` says "elevate on this host" for 8+ unrelated auth failures.
- Approve/reject/revoke forms have no submit-state feedback — double-submit possible.
- Single-challenge Reject in pending bar has no confirm dialog; "Reject All" does.
- Admin page `<title>` always `"Admin - identree"` regardless of active tab.
- "Remove User" label on the Remove Host button (copy-paste bug).
- Deploy modal title is hardcoded English (not i18n'd).
- Duplicate `class="tz-select"` attribute (browser silently ignores second one).
- Pending modal and deploy modal don't trap focus (WCAG 2.1 AA failure).
- Success banners on dashboard never auto-dismiss.
- Admin sessions view has no empty state.
- Export CSV/JSON ignores active filters.

---

## PAM/SSSD Integration

- Token cache silently non-functional in test env (empty issuer/clientID in compose).
- `common-account` fails if SSSD crashes — no SSSD watchdog.
- `access_provider` not set — disabled users still pass account checks.
- OIDC provider re-fetched on every token cache `Check()` call.
- Break-glass rotation is synchronous post-auth — 5s pause on rotation day.
- `os.Hostname()` errors silently dropped — host shows as "(unknown)" in approval UI.

---

## Zero Test Coverage on Security-Critical Paths

`internal/server/` and `internal/challenge/` — the two largest packages — have zero
test files. Untested: full `ChallengeStore` state machine, `verifySharedSecret`,
`verifyAPIKey`, `computeStatusHMAC`, `getSessionUser`/`getSessionRole`, all CSRF
logic, `loadState`/`saveStateLocked`, `requiresAdminApproval`.
