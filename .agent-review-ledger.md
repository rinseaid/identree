## Deferred Items (Require User Decision)

1. **O2: Store context propagation** — All SQLStore methods use `context.Background()` instead of request context. Refactoring to accept `context.Context` is a medium-effort change affecting all handlers. Prevents DB connection exhaustion under cancellation storms.

2. **O3: Deploy goroutine shutdown** — `runDeployJob` doesn't select on `s.stopCh`, so SSH deploy sessions continue running up to 3 minutes after SIGTERM. Requires adding context cancellation and a WaitGroup in `Stop()`.

3. **O4: Audit sink retry** — Splunk HEC and Loki sinks discard batches on HTTP failure with no retry. For a hash-chained audit trail, dropped events create gaps. Needs retry loop with backoff in `flush()`.

4. **O5: Request correlation IDs** — No request ID links log entries for a single HTTP request across the lifecycle. Needs middleware to generate/propagate IDs.

5. **O8: Health probe parallelization** — `/healthz` probes run sequentially, summing to ~6s when all time out. K8s liveness probes with <6s timeout will false-positive. Should run probes concurrently.

6. **O9: Restart race** — `handleAdminRestart` sleeps 300ms before SIGINT, which may not be enough for the response to flush. Increasing to 1s or using explicit Flush would help.
